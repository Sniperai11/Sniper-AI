# Frontend Engineering Constitution — Sniper AI Security Platform v1.0
## دستور هندسة وتصميم الواجهات الأمامية - منصة القناص الأمني للذكاء الاصطناعي

> **وثيقة معمارية ملزمة (Binding Architectural Reference)**  
> يمثل هذا الدستور المرجع السيادي والنهائي لتصميم وتطوير وتأمين كافة الواجهات الرسومية، المكونات، وتجربة المستخدم (UI/UX) لمنصة **Sniper AI Security**. يجب إخضاع جميع التحديثات البرمجية وعمليات إعادة الهيكلة والتطوير في الطبقة العرضية (Presentation Layer) للتوافق التام مع المعايير الواردة في هذه الوثيقة لضمان اتساقها مع معمارية الأنظمة الأمنية الدولية الكبرى (مثل CrowdStrike Falcon و Palo Alto Cortex).

---

## 1. الرؤية الهندسية وفلسفة التصميم (Design Philosophy)

### 1.1 من لوحة بيانات عادية إلى مساحة عمل مؤسسية (Workspace vs Dashboard)
منصة Sniper AI ليست مجرد موقع لعرض الإحصائيات (SaaS Metrics)، بل هي **مساحة عمل متكاملة لمحللي الأمن ومستشاري الاختراق (Enterprise Security Workspace)**.
* **التركيز البؤري (Information Density):** استغلال المساحات الرسومية بكفاءة عالية لعرض البيانات الحرجة دون إحداث تشتيت بصري.
* **الوضوح الفوري (Instant Clarity):** إبراز مستويات الخطورة، وحالة الفحص الجاري، وتوصيات الذكاء الاصطناعي بشكل فوري دون الحاجة للتنقل العميق.
* **التكامل بين السلوك والبيانات:** تصميم واجهات تفاعلية متزامنة فورياً مع الخادم عبر بروتوكولات مثل Server-Sent Events (SSE) أو Websockets لتتبع الفحوصات والترميم الذاتي تلقائياً.

---

## 2. المبادئ الدستورية للواجهات (Constitutional UI Principles)

### 2.1 الأولوية للمؤسسات والأمن (Enterprise & Security First)
* **التحكم المستند إلى الأدوار (RBAC Visual States):** تخصيص العناصر المرئية ومفاتيح التحكم بناءً على رتبة المستخدم الحالية (مسؤول النظام Admin، خبير فحص Hunter، أو مستعرض Viewer).
* **مناطق الخطر والحوارات التحذيرية (Danger Zones & Confirmation Dialogs):** أي إجراء تدميري (مثل إزالة عضو فريق، أو تعطيل 2FA، أو حذف مشروع) يجب أن يمر عبر نافذة تأكيد مدعومة بتأكيد نصي أو رمز تحقق ثنائي لتجنب الأخطاء البشرية.
* **إظهار سلامة وحالة الحساب:** عرض دائم لمستوى حماية الحساب (على سبيل المثال، حالة تفعيل Firebase Auth 2FA) لضمان التزام مستخدمي المنصة بالمعايير الأمنية العليا.

### 2.2 الهوية المعتمدة على الذكاء الاصطناعي (AI-Native Interface)
* الذكاء الاصطناعي في المنصة ليس مجرد نافذة محادثة (Chat Tab) معزولة، بل جزء لا يتجزأ من كل شاشة:
  * **توصيات الترميم الذاتي (AI Remediation Proposals):** عرض الأكواد المقترحة والحلول الأمنية (مقارنة الكود المصاب بالكود النظيف) بشكل تفاعلي.
  * **معاملات الثقة (Confidence Scores):** إرفاق نسبة مئوية ومستوى ثقة لكل تحليل صادر عن محرك Gemini لضمان دقة اتخاذ القرار.
  * **ملخصات الرصد والخطورة التنفيذية:** تلخيص فوري للثغرات المكتشفة بنقرة زر واحدة لتسهيل إعداد تقارير الإدارة.

## 3. المعمارية الهيكلية للمكونات (Component Architecture)

تعتمد المنصة منهجية **التصميم الذري (Atomic Design)** المقسمة بوضوح إلى طبقات معزولة:

```
src/
├── components/          # المكونات الذرية الصغيرة (Atoms & Molecules)
│   ├── ui/              # المكونات الأساسية المشتركة (Buttons, Inputs, Badges, Modals)
│   └── custom/          # المكونات المخصصة للمجال الأمني (SecurityTerminal, CodeDiff)
├── pages/               # الصفحات الكاملة واللوحات المستقلة (Dashboard, Workspace, Compliance)
├── hooks/               # خطافات تفاعلية مخصصة لإدارة العمليات ومزامنة البيانات
├── services/            # الخدمات الموجهة للتواصل مع الخلفية (API Clients, Firebase Auth)
├── types/               # إعلانات الأنواع والواجهات الثابتة في TypeScript
└── utils/               # توابع المساعدة والتنسيق المستقلة
```

### 3.1 قواعد ذهبية لتطوير المكونات:
1. **المسؤولية الفردية (Single Responsibility):** كل مكون مسؤول عن وظيفة واحدة محددة بوضوح.
2. **استقرار الـ Effects وعلاقات الاعتمادية:** تجنب Re-renders اللانهائية في `useEffect`. لا يتم تضمين كائنات أو مصفوفات غير مستقرة في مصفوفة الاعتماديات، بل نعتمد على القيم الأولية (Primitives) لضمان ثبات الأداء.
3. **أمان المتغيرات البيئية:** عدم تمرير أي مفاتيح سرية (مثل `GEMINI_API_KEY`) للطبقة العرضية، والاعتماد المطلق على طلبات خادم Express الخلفي كـ Proxy آمن.

## 4. نظام الهوية البصرية والتصميم (Visual Design System)

### 4.1 الألوان والسمات (Theme & Color Tokens)
تعتمد المنصة على نمط **Dark Cosmic Slate** الفاخر والمستوحى من أنظمة حماية غرف العمليات السيبرانية:

* **الخلفية الأساسية (Base Background):** لون رمادي داكن مشبع بدرجة منخفضة جداً من درجات الأزرق الكوني والرمادي (`#090d16` أو `Slate-950`).
* **مستويات التباين والأبعاد (Contrast Limit):** يجب ألا يتجاوز الفرق اللوني البصري بين الحاويات (Containers) والخلفيات المستند عليها حاجز الـ 12% لضمان راحة أعين المحللين في فترات المراقبة الطويلة.
* **الألوان الدلالية للأمن (Semantic Security Colors):**
  * **خطورة حرجة (Critical / High):** درجات الأحمر القاني (`#ef4444` / `Red-500` أو `Rose-600`) ترمز إلى الثغرات الواجب معالجتها فوراً.
  * **خطورة متوسطة (Medium):** درجات البرتقالي الكهرماني (`#f59e0b` / `Amber-500`) ترمز للتحذيرات والمراجعات.
  * **خطورة منخفضة (Low):** درجات الأصفر الفاتح أو الرمادي الفضي.
  * **آمن / تم التحقق (Secure / Passed):** الأخضر الزمردي المتألق (`#10b981` / `Emerald-500`).

### 4.2 التنسيق المطبعي والهيكل الشجري (Typography & Layout)
* **الخطوط الأساسية:** استخدام خطوط ذات تباين وهندسة مريحة (مثل Cairo أو Plus Jakarta Sans) ومطابقتها التامة لتقسيمات العنوان والفقرات دون تخطي مستويات الهرم المطبعي (H1 -> H2 -> H3).
* **حجم الخطوط للمحتويات التفاعلية:** منع التفاف العبارات أو تداخل الحروف داخل الأزرار والشارات الصغيرة (Badges & Pills)، مع ضمان حد أدنى لحجم النص البرمجي يبلغ `12px` مدعوماً بخط أحادي المسافة (Monospace font) لقراءة شفرات الكود المصدرية.

## 5. مصفوفة التحقق الثلاثي والترميم التلقائي (Auto-Remediation Compliance)

بموجب المجلد الثاني عشر من الدستور العام (Volume XII):
* واجهة الترميم التلقائي يجب أن تستعرض وبشكل حيوي تقدم عملية المحاكاة البرمجية المعزولة (Sandboxed Sandbox Environment).
* عرض الأكواد يجب أن يتم بصيغة مقارنة مرئية واضحة وجذابة (Before/After) تعكس:
  1. شفرة الكود المصابة المكتشفة من قبل محرك الفحص الذكي للثغرات (Vulnerable).
  2. شفرة الكود النظيفة والمصلحة والمطهرة (Patched) المسترجعة بعد معالجتها بواسطة نموذج الذكاء الاصطناعي الموجه.
  3. سجلات التجميع والفحص والتحقق الثلاثي المعتمدة على الخادم لتبديد الشكوك وبناء جدار الثقة لدى المهندسين الأمنيين.

---

## 6. إقرار التوافق الهيكلي والتأثير الأمني (Architectural Agreement)

يلتزم كافة مطوري الواجهة وفريق هندسة الأنظمة بمطابقة كل التحديثات المرئية مع معايير هذا الدستور، وتعتبر أي واجهة لا تلبي معايير الكفاءة العالية، أو تتسبب في تسريب بيانات سرية للعميل، أو تعرض شاشات مشتتة وخلفيات ذات تباين ضعيف، خرقاً صريحاً للمبادئ التشغيلية لمنصة **Sniper AI Security**.
